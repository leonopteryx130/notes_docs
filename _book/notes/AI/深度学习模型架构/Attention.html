Attention

基本思想

注意力的本质：用 Query 去查一组 (Key, Value)，按相似度加权取出 Value。

角色 含义 直觉
Query (Q) 当前想「问」什么 此刻的问题 / 解码器当前状态
Key (K) 用来被比对的「索引」 每条候选信息的标签
Value (V) 真正要取出的内容 与 Key 对应的信息本体

计算流程统一为三步：

  1. 打分：对每个 Key，算它与 Query 的相似度 si=F(Q,Ki)s_i = F(Q, K_i)s​i​​=F(Q,K​i​​)
  2. 归一化：把分数变成权重（通常 Softmax），αi=softmax(s)i\alpha_i = \mathrm{softmax}(s)_iα​i​​=softmax(s)​i​​，且 ∑iαi=1\sum_i \alpha_i = 1∑​i​​α​i​​=1
  3. 加权求和：用权重聚合 Value，得到注意力输出

Attention(Q,K,V)=∑iαiVi\mathrm{Attention}(Q, K, V) = \sum_i \alpha_i\, V_iAttention(Q,K,V)=∑​i​​α​i​​V​i​​

常见的相似度函数 FFF：点积 Q⊤KQ^\top KQ​⊤​​K、缩放点积 Q⊤K/dQ^\top K / \sqrt{d}Q​⊤​​K/√​d​​​、余弦相似度，以及 Bahdanau 的加性注意力。

Key 与 Value 可以相同（很多序列模型里都是），也可以不同；一般有多少个 Key，就有多少个对应的 Value。
